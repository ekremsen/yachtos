# Yacht UI Design

**Version:** 1.0  
**Status:** Draft  
**Document Owner:** Product Team  
**Last Updated:** 29 June 2026

---

## Purpose

This document defines the user interface requirements for the Yacht module.

The Yacht interface should provide quick access to yacht information while acting as the central navigation point for all operational activities.

---

## Design Principles

- Simple
- Fast
- Mobile-first
- Operationally focused
- Easy to learn
- Minimal clicks

---

## Yacht List

The yacht list should display:

- Yacht name
- Yacht type
- Home port
- Flag country
- Status

Available actions:

- View
- Edit
- Archive

---

## Yacht Detail

The yacht detail page should present the yacht as the operational hub.

Sections may include:

- General Information
- Assigned Members
- Maintenance Summary
- Inventory Summary
- Active Tasks
- Guest Preferences
- Documents

Each section should provide quick navigation to its corresponding module.

---

## Yacht Create

The yacht creation page should request only the minimum required information.

Required:

- Yacht Name

Optional:

- Registration Number
- IMO Number
- Hull Number
- Flag Country
- Home Port
- Yacht Type
- Manufacturer
- Model
- Build Year

The user should be able to complete yacht creation within one minute.

---

## UX Rules

- Minimize typing whenever possible.
- Use dropdowns for standardized values.
- Group related information together.
- Avoid overwhelming users with optional fields.
- Prioritize frequently used information.

---

## Mobile Requirements

The Yacht module must be fully usable on mobile devices.

Captains should be able to:

- View yacht information
- Update yacht details
- Access operational modules

without requiring a desktop device.
